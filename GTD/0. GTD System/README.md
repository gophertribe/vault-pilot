# GTD System Overview

This vault follows the Getting Things Done (GTD) methodology by David Allen.

## Folder Structure

- **0. GTD System/**: System documentation and templates
- **1. Inbox/**: Capture everything here first
- **2. Next Actions/**: Actionable items organized by context
- **3. Projects/**: Multi-step outcomes requiring more than one action
- **4. Someday Maybe/**: Things you might want to do someday
- **5. Reference/**: Information you might need later
- **6. Weekly Reviews/**: Weekly planning and review sessions

## Workflow

1. **Capture**: Put everything in Inbox
2. **Clarify**: Process inbox items - what is it? Is it actionable?
3. **Organize**: Put items in appropriate folders
4. **Reflect**: Weekly reviews to stay current
5. **Engage**: Do the work with confidence

## Quick Start

1. Use templates from `0. GTD System/Templates/`
2. Process inbox to zero weekly
3. Review projects and next actions weekly
4. Keep reference materials organized and searchable

# GTD System Overview

This folder contains the core GTD system documentation and templates.

## Folder Structure
- **0. GTD System/** - System documentation and templates
- **1. Inbox/** - Capture bucket for all new items
- **2. Next Actions/** - Organized by context (@calls, @computer, etc.)
- **3. Projects/** - Active, template, and archived projects
- **4. Someday Maybe/** - Items for future consideration
- **5. Reference/** - Reference materials and documentation
- **6. Weekly Reviews/** - Weekly review records

## Quick Start
1. Capture everything in Inbox
2. Process inbox items weekly
3. Organize next actions by context
4. Review projects regularly
5. Conduct weekly reviews

## Key GTD Principles
- Capture everything
- Clarify what it means
- Organize by context and priority
- Reflect through regular reviews
- Engage with confidence
